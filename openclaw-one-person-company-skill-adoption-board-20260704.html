Evidence: Skills are markdown instruction files that teach an agent how and when to use tools; workspace skills have highest precedence.

Team rule: Use local workspace skills for one-person company operating behavior before any external install or runtime extension.

OpenClaw Plugins

Evidence: Plugins extend OpenClaw with runtime capabilities such as channels, model providers, tools, skills, hooks, and media features.

Team rule: Do not add runtime capability while runtime hooks and memory pressure remain unresolved.

ClawHub

Evidence: ClawHub is the public registry for OpenClaw skills and plugins, with public pages and security scan summaries for inspection.

Team rule: Treat every third-party registry package as untrusted until Support/Ops reviews source, scan state, publisher, scope, and rollback.

Skills config

Evidence: OpenClaw supports agent skill allowlists and skill entry configuration.

Team rule: Use allowlist thinking: expose only the one-person company operator skill needed for current department work.

Creating skills

Evidence: Custom skills require a SKILL.md file with name and description frontmatter.

Team rule: Keep the local skill small, explicit, frontmatter-valid, and free of dependency installers or secret handling.

部門任務

CEO/Strategy

Approve the local workspace skill as the current internal operating layer.

  • Treat the local one-person company operator skill as the preferred skill route because it does not add runtime capability.
  • Reject new plugin work until Support/Ops proves memory, hooks, and rollback are stable.
  • Route non-external department questions back to Codex decision, not the human owner.

Support/Ops

Protect OpenClaw from unsafe installs, runtime hooks, secrets, and external authority drift.

  • Do not install third-party ClawHub skills or plugins automatically.
  • Require source review, scan state, publisher check, scope check, allowlist decision, and rollback path for any future package.
  • Block runtime plugins while OpenClaw runtime remains memory-risky or hook-prone.

Product Studio QA

Validate that the workspace skill improves workflow without expanding user-facing product claims.

  • Confirm the skill points Product Studio QA to Mood Anchor RWD, offline, localStorage, and safety checks.
  • Confirm no medical, crisis, revenue, traffic, or customer outcome promise appears in the skill.
  • Confirm the skill remains small enough to avoid unnecessary prompt overhead.

Analytics

Track adoption evidence without scraping or sensitive data.

  • Record owner-entered aggregate only: skill changes, review outcomes, rollback readiness, and owner-wait avoided counts.
  • Do not scrape package registries, private dashboards, agent logs, secrets, customer messages, or platform analytics.
  • Recommend keep, revise, or rollback from local review counts only.

Marketing Desk

Use the skill only for internal positioning and channel-safe draft routing.

  • Prepare no public copy about OpenClaw automation unless Support/Ops approves exact wording.
  • Keep Mood Anchor and Listing Audit channel drafts owner-reviewed and manual-only.
  • Do not publish, post, submit, schedule, DM, email, or claim autonomous business results.

停止線

  • Do not install third-party ClawHub skills or plugins automatically.
  • Do not add a runtime plugin while OpenClaw runtime remains memory-risky or hook-prone.
  • Do not use OpenClaw message commands for team control while message surfaces can spawn hooks.
  • Do not request, store, print, or inject secrets, tokens, chat ids, account credentials, payout data, tax data, or private analytics.
  • No external authority actions: login, posting, public replies, DMs, email, checkout, pricing, refunds, payout, tax, credentials, account settings, contracts, or real-world commitments.
  • No revenue, traffic, ranking, customer, safety, clinical, crisis, or health-outcome promises.
  • Analytics remains owner-entered aggregate only.
